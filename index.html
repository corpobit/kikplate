Kikplate Helm Chart

Deploy Kikplate on Kubernetes with a single command. A production-ready Helm chart for managing self-hosted boilerplate templates.

Get Started → View on GitHub

Quick Start

1. Add the Helm Repository

Add the Kikplate Helm repository to your Helm configuration:

helm repo add kikplate https://moeidheidari.github.io/kikplate/charts

Then update your repositories:

helm repo update

2. Install the Chart

Install Kikplate into your Kubernetes cluster:

helm install kikplate kikplate/kikplate \ --namespace kikplate \ --create-namespace

Customize the installation with your own values:

helm install kikplate kikplate/kikplate \ -f values.yaml \ --namespace kikplate

3. Configure Secrets

Set your OAuth provider credentials and other secrets:

helm install kikplate kikplate/kikplate \ --set secrets.jwtSecret=YOUR_JWT_SECRET \ --set secrets.sso.githubClientSecret=YOUR_GITHUB_SECRET

4. Access the Application

Port forward to access the application locally:

kubectl port-forward -n kikplate \ svc/web 3000:3000

Then visit http://localhost:3000

Features

🚀 Production Ready

Fully configured with health checks, resource limits, and best practices for Kubernetes deployments.

🔐 Security First

Secrets management, environment variable configuration, and secure credential handling out of the box.

🔧 Highly Customizable

Flexible configuration through values.yaml for all deployment parameters and application settings.

💾 Database Included

Includes PostgreSQL with persistent volumes for reliable data storage in Kubernetes.

🌐 Multi-Component

Manages API, Web, Sync worker, PostgreSQL, and supporting services in a single helm install.

📊 Monitoring Ready

Built-in liveness and readiness probes for automatic pod health monitoring and recovery.
